## 二、dirty 标记如何存储

### 存储位置

```
data/index/dirty-files.json
```

### 数据格式

```json
[
  {
    "bookId": "demo-guixu",
    "path": "人物设定/林晓.md",
    "updatedAt": "2026-05-24T12:34:56.789Z"
  }
]
```

### 触发时机

每次通过 `writeBookFile()` 写入文件后，自动调用 `markDirty(bookId, filePath)`：
- 如果该 `(bookId, path)` 已存在，更新 `updatedAt`
- 如果不存在，追加新条目

### 服务端封装

所有 dirty 逻辑封装在 `lib/server/dirty-index.ts`，组件层不直接操作文件路径：

| 函数 | 用途 |
|------|------|
| `markDirty(bookId, filePath)` | 标记文件为 dirty |
| `getDirtyFiles(bookId?)` | 查询 dirty 文件，可按 bookId 过滤 |
| `clearDirty(bookId, filePath)` | 清除单个 dirty 标记（供后续阶段使用） |
