## 一、产品定位

**一句话**：面向创作者的 AI 书籍系统管理工作台，核心是「系统 Agent」而非「写作 Agent」。

```
写作 Agent → 官网更好，我们不抢
系统 Agent → 我们的主场：世界观/人物/情节/设定的增删改查与一致性维护
```

### 2.2 系统分层

```
Layer 0  用户输入
    ↓
Layer 1  意图理解循环（最多 3 轮）
    ↓
Layer 2  ActionPlan 生成 & 用户确认
    ↓
Layer 3  程序执行（无 LLM）
    ↓
Layer 4  Ledger 记录 & 状态更新
```

## 三、核心流程：意图理解循环

### 3.1 流程图

```
用户输入
    ↓
[Observe]   原始输入解析，识别显式指令 & 隐含需求
    ↓
[Hypothesize] 意图假设（可能多个，分开列）
    ↓
[Retrieve]  多路召回（见 3.2）
    ↓
[Ground]    阅读召回证据，检验假设
    ↓
[Revise]    修正意图 → 不确定则发起澄清（最多 3 轮）（1s 后自动生成actionplan，可随时终止）
    ↓
[Plan]      生成 ActionPlan
    ↓
[Display]   展示给用户
    ↓
[Act]       程序执行，LLM 不再介入
```

### 3.2 多路召回策略

| 召回类型   | 作用                       | 举例                       |
| ---------- | -------------------------- | -------------------------- |
| 向量召回   | 语义相似内容               | "这个角色之前说过类似的话" |
| 关键词召回 | 精确名词、章节名、人名     | "第三章"、"林晓"           |
| 规则召回   | 强制拿目标文件、最近上下文 | 当前书籍设定文件           |
| 图谱召回   | 人物关系、事件链           | "林晓的师父是谁"           |
| 时间召回   | 最近对话或文件修改         | 上一次改动的设定           |

### 3.3 意图多路并行处理

```
用户输入可能包含多个混杂意图，必须拆分：

示例输入："把林晓改成女的，顺便把她和陈磊的关系变成敌对，
           第五章好像也提到他们了"

拆分后：
  意图A：人物属性修改 → 林晓.gender = female
  意图B：关系修改    → 林晓 <-> 陈磊 = 敌对
  意图C：章节影响    → 第五章需要同步检查
```

## 四、ActionPlan 展示系统（右栏）

### 4.1 Plan 展示卡片（仿 Codex）

用户确认前，显示 4 个维度：

```
┌─────────────────────────────────────┐
│ 📋 我理解的任务                      │
│   - 修改林晓性别为女                  │
│   - 更新林晓与陈磊关系为敌对           │
│                                     │
│ 📂 我会使用的上下文                   │
│   - 人物设定/林晓.md                  │
│   - 人物关系图谱                      │
│   - 第五章.txt（关联检查）             │
│                                     │
│ 🚧 我不会改动的边界                   │
│   - 第一章至第四章正文                 │
│   - 其他人物设定                      │
│                                     │
│ ⚠️ 我发现的缺失信息                   │
│   - 林晓在第七章还有出现，是否同步？    │
│                                     │
│   [确认执行]  [修改计划]  [放弃]       │
└─────────────────────────────────────┘
```

### 4.2 引用卡片（右栏 → 对话）

右栏「设定快览」区的任意卡片（人物/地点/事件）可直接引用到对话输入框，方便用户精确指定操作对象。

```
右栏卡片：
┌──────────────────┐
│ 👤 林晓           │
│ 性别：男 / 武者    │
│ 关系：陈磊（友）   │
│  [引用到对话] →   │
└──────────────────┘

引用后，输入框自动插入：
  @林晓 [人物设定/林晓.md]
```

### 4.3 ActionPlan 聚合树结构

```typescript
interface ActionNode {
  id: string
  type: 'group' | 'action'
  label: string
  
  // 作用域
  scopePath: string          // e.g. "人物设定/林晓.md"
  targetBlock?: string       // 具体 block id
  affectedChapters?: string[]
  
  // 操作
  operation: 'create' | 'update' | 'delete' | 'check'
  
  // 聚合信息（group 节点用）
  childCount?: number
  riskLevel?: 'low' | 'medium' | 'high'
  status: 'pending' | 'running' | 'done' | 'failed'
  
  children?: ActionNode[]
}
示例树：

▼ 修改林晓角色 [3项] [风险:中]
  ✓ 更新 人物设定/林晓.md → gender: female
  ✓ 更新 关系图谱 → 林晓-陈磊: 敌对
  ⚡ 检查 第五章.txt → 扫描相关描写

▼ 待确认项 [1项]
  ? 第七章.txt → 是否同步修改？
```

### 4.4 执行后状态

```
用户确认 → 程序接管：
  1. 校验 ActionCard 合法性
  2. 定位目标 Block
  3. 执行替换/创建/删除
  4. 写入 Ledger（时间戳、操作内容、操作前快照）
  5. 卡片标记为 Done，1s 后收起

LLM 不再参与执行阶段。
```

## 五、原生写作台

### 5.1 触发方式

点击左栏章节树中的任意章节节点 → 中栏切换为写作台模式，右栏保持不动（仍显示 ActionPlan / 设定快览）。

> 写作台不替换右栏，只替换中栏。左栏章节树底部显示「← 返回对话」按钮退出。

### 5.2 中栏布局（写作台模式）

```
┌──────────────────────────────────────────┐
│  工具栏：B  I  H1  H2  ─  链接  图片  …  │
│  章节：第二章  |  字数：2,847 字           │
├──────────────────────────────────────────┤
│                                          │
│  富文本编辑区（上层，~65% 高度）           │
│                                          │
│  正文正文正文...                          │
│                                          │
│  支持：段落样式 / 粗体斜体 / 标题层级     │
│        字数统计 / 排版预览 / 章节跳转     │
│                                          │
├──────────────────────────────────────────┤
│  💭 试写沙盒（下层，~35% 高度）            │
│                                          │
│  [AI 试写内容，临时区，不写入设定文件]     │
│                                          │
│  [保留到草稿]  [扔掉]  [继续生成]         │
└──────────────────────────────────────────┘
```

### 5.3 富文本编辑器功能范围

| 功能     | 说明                                      |
| -------- | ----------------------------------------- |
| 段落样式 | 正文、标题一至三、引用块                  |
| 行内样式 | 粗体、斜体、下划线、删除线                |
| 字数统计 | 实时显示当前章节字数 & 全书总字数         |
| 章节管理 | 在写作台内新建/删除/拖拽排序章节          |
| 排版预览 | 切换「编辑」/「预览」视图，预览仿印刷排版 |
| 自动保存 | 编辑后 2s 自动保存，写入对应章节文件      |

### 5.4 试写沙盒规则

- 内容**不写入**任何设定文件，也不触发 ActionPlan
- 用户主动「保留到草稿」才写入富文本编辑区
- 「扔掉」直接清空，无需确认
- 提示词复用 Gemini 同一套（形成对比数据 → 后续微调素材）

## 六、建书流程

### 6.1 建书模式

「新建书籍」不是引导问答流程，而是一个**系统状态标记**：进入建书状态后，Agent 的意图理解上下文切换为「初始化书籍档案」模式，用户自由描述，系统按建书语义来解析输入、生成档案结构。

```
触发：点击「+ 新建书籍」
系统行为：
  → 标记当前会话状态 = BOOK_INIT
  → 创建空书籍档案（书名待定）
  → Agent 提示用户可以描述书籍，但不强制引导

用户可以：
  - 粘贴已有设定
  - 随意描述故事梗概
  - 直接说"帮我建一本奇幻小说的空架子"

Agent 根据输入自动判断：
  → 生成初始档案结构
  → 创建默认文件夹
  → 退出 BOOK_INIT 状态，进入正常对话模式
```

### 6.2 书籍档案结构

```
📚 书名/
├── 人物设定/
│   ├── 林晓.md
│   └── 陈磊.md
├── 世界观/
│   ├── 地图.md
│   └── 规则体系.md
├── 关系图谱.json
├── 章节大纲/
│   ├── 第一章.md
│   └── ...
├── 创作指南.md（可选引入）
└── ledger.jsonl  ← 操作日志
```

## 七、工作台（独立页面）

工作台为单独页面，不与主对话界面共存，通过顶部导航或快捷键跳转进入。

### 7.1 功能范围

```
工作台 = 所有设定的可视化编辑界面

用户可以：
  - 直接编辑任何设定文件
  - 查看修改时间戳（用于上下文优先级排序）
  - 查看关系图谱可视化
  - 浏览 Ledger 历史（可回滚）
```

### 7.2 页面结构

```
┌─────────────────────────────────────────────────┐
│  顶部导航：[对话] [工作台 ←当前] [设置]           │
├──────────────┬──────────────────────────────────┤
│  文件树       │  编辑器区                         │
│  ──────────  │  ──────────────────────────────  │
│  📁 人物设定  │  林晓.md                          │
│    林晓.md   │                                   │
│    陈磊.md   │  [Markdown 编辑器]                │
│  📁 世界观    │                                   │
│  📁 章节大纲  │  ──────────────────────────────  │
│              │  相关内容面板                      │
│  ledger ↩    │  当前文件关联的其他设定             │
└──────────────┴──────────────────────────────────┘
```

### 7.3 修改时间戳的用途

每次召回时，时间召回策略会优先考虑最近修改的设定文件，越新的改动在上下文中权重越高。Agent 的认知随用户工作自然更新，无需手动刷新上下文。

## 八、创作指南机制

```
创作指南.md = 用户习惯 & 偏好的沉淀文件

暂时方案：手动维护
未来方向：skill 积累系统（本期不做）

在建书时可选择是否引入。
引入后作为系统上下文的固定部分，
每次对话都会携带（token 成本需控制，考虑摘要压缩）。
```
