## 二、Skill 数据结构

```typescript
interface Skill {
  id: string                    // 唯一 ID，如 "skill-style-demo-guixu"
  type: "style_guide"          // 本期只有这一种
        | "plot_pattern"       // 未来：惯用情节结构
        | "char_template"      // 未来：人物创建模板
        | "world_rule"         // 未来：世界观通用规则
  scope: "global" | "book"     // 跨书籍 / 仅当前书籍
  bookId?: string              // scope = book 时必填
  sourceFile: string           // "创作指南.md"
  summaryFile: string          // "skills/style_guide_summary.md"
  summaryTokenCount: number    // 摘要 token 估算
  lastSourceModified: string   // 源文件最后修改时间（ISO）
  lastSummaryGenerated: string // 摘要最后生成时间（ISO）
  dirty: boolean               // 源文件比摘要新 → 需要刷新
}
```

### 元数据存储位置

```
data/books/{bookId}/skills/style_guide.skill.json
```

## 三、创作指南摘要如何生成

### 规则摘要逻辑（不接 LLM）

1. 读取 `创作指南.md` 全文
2. 逐行扫描：
   - **h1/h2 标题**：始终保留
   - **关键词段落**：如果当前 h2 标题包含 `文风`、`语感`、`禁忌`、`人物`、`结构`、`节奏`、`偏好`、`塑造`、`风格`、`写法` 等关键词，该段落下的所有行都保留
   - **散落关键词行**：任何包含上述关键词的行，即使不在匹配段落下也保留
3. 如果没有任何行被选中，取源文件前 8 行非空行作为兜底
4. 输出控制在约 500 中文字符以内
5. 如果源文件为空，写入占位摘要：`（创作指南为空,请在工作台编辑创作指南.md）`

### 输出格式

```markdown
# 创作指南摘要

## 语感

保持冷峻、克制、近物质的语感。少用感叹号,多用短句。

## 人物塑造

林晓：寡言,行动多于言语。内心独白用环境描写暗示。

## 禁忌

- 不用"突然"开头
- 不用过多形容词堆砌
- 不写说教式旁白
```

### Token 估算

粗略按 `字符数 / 1.5` 估算中文 token 数。

## 四、dirty 如何判断

### 判断流程

```
getStyleGuideSkill(bookId)
  → 读取 skills/style_guide.skill.json（metadata）
  → 读取 创作指南.md 的 mtime（getBookFileMtime）
  → 如果 metadata 不存在 → dirty = true（首次）
  → 如果 sourceMtime > metadata.lastSourceModified → dirty = true
  → 否则 dirty = false
```

### 刷新后状态

```
refreshStyleGuideSummary(bookId)
  → 读取 创作指南.md 内容
  → generateSummary(content)
  → 写入 skills/style_guide_summary.md
  → 更新 metadata:
    - lastSourceModified = 源文件当前 mtime
    - lastSummaryGenerated = 当前时间
    - summaryTokenCount = 估算值
    - dirty = false
  → 写入 skills/style_guide.skill.json
```

## 六、messages API 如何注入摘要

### 接入位置

`POST /api/books/{bookId}/messages` 中，在召回之后读取摘要：

```
用户发送消息
  → retrieveContext(bookId, content)        // 召回相关上下文
  → readStyleGuideSummary(bookId)           // 读取创作指南摘要
  → 如果摘要存在，contextPaths.push("skills/style_guide_summary.md")
  → 意图解析
  → 生成 ActionPlan
  → 组装 assistant message（含 brief.contextPaths）
  → 返回 { message, plan }
```

### 注入规则

- 只注入路径 `skills/style_guide_summary.md`，不注入完整摘要内容
- 摘要内容在后续 LLM 接入时作为 Layer A 固定层注入（本期不做）
- 如果摘要为空，不注入
- 读取失败不阻塞消息处理（`.catch(() => "")` ）
- 不把完整 `创作指南.md` 塞入 message 或 brief

### Skill dirty 时的行为

- 不阻塞当前对话
- dirty 状态通过 Skill API 暴露，工作台 Skill tab 可见"需要刷新"标签
- 用户可手动点击"刷新摘要"按钮
