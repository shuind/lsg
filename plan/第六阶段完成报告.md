## 二、RetrievedContext 数据结构

```typescript
interface RetrievedContext {
  id: string           // 唯一 ID，格式: rc-{index}
  bookId: string       // 所属书籍 ID
  path: string         // 文件相对路径，如 "人物设定/林晓.md"
  reason: "dirty" | "keyword" | "recent"  // 召回原因
  score: number        // 相关性评分（越高越相关）
  updatedAt: string    // 文件最后修改时间
  excerpt: string      // 文件摘要，截断到 ~200 中文字符
}
```

## 三、retrieveContext 召回规则

### 评分体系

| 来源 | 分数 | 说明 |
|------|------|------|
| dirty 文件 | +100 | 最近被修改过的文件，最高优先级 |
| 文件名/路径含关键词 | +50 | 文件名或路径匹配 query 中提取的关键词 |
| 文件内容含关键词 | +30 | 文件内容匹配 query 中的关键词 |
| 1 小时内修改 | +10 | 时间衰减加分 |
| 24 小时内修改 | +5 | 时间衰减加分 |

### 关键词提取

从 query 中提取三类关键词：

1. **人名**：匹配 `把X改成`、`把X和Y关系` 模式中的 X、Y（2-10 个中文字符）
2. **路径片段**：如果 query 包含 `创作指南`、`关系图谱`、`世界观`、`人物设定`、`章节大纲`、`章节`、`skills` 等
3. **通用中文词**：2-4 个连续中文字符（排除常见停用词）

### 排序规则

1. 先按 score 降序
2. 同分时按 updatedAt 降序

### 返回数量

最多返回 5 条。

### 跳过规则

以下文件默认跳过（除非 query 明确包含其名称）：
- `ledger.jsonl`
- `pending-action-plan.json`
- `book.json`

### excerpt 生成

- 如果匹配到关键词，截取关键词前后约 200 字符
- 否则取文件前 200 字符
- 换行替换为空格

## 四、dirty 文件如何参与召回

1. `retrieveContext` 调用 `getDirtyFiles(bookId)` 获取当前书籍的 dirty 文件列表
2. dirty 文件直接获得 +100 分基础分（最高优先级）
3. dirty 文件的 `updatedAt` 来自 dirty-index（反映被标记时间）
4. **不清除 dirty**：本阶段只是读取 dirty 文件用于召回，不调用 `clearDirty`
5. dirty 文件即使不在文件树中（如已被删除），也会被加入候选（从 dirty-index 补充）
6. 后续"索引重建完成"阶段再清除 dirty

## 六、messages API 如何使用召回结果

### 接入位置

`POST /api/books/{bookId}/messages` 中，在意图解析之前先调用 `retrieveContext`：

```
用户发送消息
  → retrieveContext(bookId, content)    // 召回相关上下文
  → 意图解析（正则匹配）
  → 生成 ActionPlan
  → 组装 assistant message（含 brief.contextPaths）
  → 返回 { message, plan }
```

### assistant message 变化

| 场景 | message.content | message.brief |
|------|----------------|---------------|
| 有 plan + 有 context | "已根据最近改动和相关设定拟好变更计划,请在右侧确认。" | `{ understood: [], contextPaths: [...] }` |
| 有 plan + 无 context | "已为你拟好变更计划,请在右侧确认。" | undefined |
| 无 plan + 有 context | "我找到了相关上下文,但暂时没有生成可执行计划。" | `{ understood: [], contextPaths: [...] }` |
| 无 plan + 无 context | "已收到,正在意图分析..." | undefined |

### contextPaths 来源

直接使用召回结果的 `path` 数组，格式如 `["人物设定/林晓.md", "关系图谱.json"]`。

### 容错

- `retrieveContext` 调用被 `.catch(() => {})` 包裹，召回失败不影响消息处理
- 文件读取异常不会崩溃 messages API
