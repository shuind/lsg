## 二、新建书籍交互流程

```
用户点击左栏"+ 新建书籍"
  ↓
window.prompt 弹出输入框："请输入书名"
  ↓
用户输入书名 → 调用 createBook(title)
  ↓  （用户取消或输入空 → 不创建，直接返回）
POST /api/books { title }
  ↓
BookStore.createBook() 在 data/books/ 下创建：
  {bookId}/
  ├── book.json          （id, title, createdAt, updatedAt）
  ├── 创作指南.md
  ├── 关系图谱.json
  ├── ledger.jsonl
  ├── 人物设定/
  ├── 世界观/
  ├── 章节大纲/
  └── skills/
  ↓
调用 listBooks() 从 API 重新获取完整书籍列表
  ↓
setBooks(bs)           — 用完整列表替换前端状态
setActiveBookId(b.id)  — 自动选中新创建的书
setActiveChapterId(null) — 清空章节选中
setMode("chat")        — 回到对话模式
setWorkbenchBookId(null) — 关闭工作台（如果打开的话）
```

### 错误处理

- 用户取消 prompt 或输入空字符串 → 不创建，静默返回
- API 调用失败 → `catch` 捕获，`alert("创建书籍失败，请重试")`，不会白屏

## 三、创建后页面状态如何更新

| 状态 | 更新方式 |
|------|----------|
| `books` | 从 API 重新获取完整列表（而非前端拼接），保证与服务端一致 |
| `activeBookId` | 设为新创建的书的 id |
| `activeChapterId` | 清空为 `null` |
| `mode` | 切回 `"chat"` |
| `workbenchBookId` | 设为 `null`（关闭工作台） |
| `chapters` / `messages` / `actions` / `cards` | 由 `activeBookId` 变化的 useEffect 自动重新加载 |

### activeBookId 初始值修正

**问题**：原来 `activeBookId` 初始值硬编码为 `"b1"`，当真实 API 返回的书籍 id 不是 `"b1"` 时，初始选中状态不一致。

**修复**：初始值改为空字符串。书籍列表加载完成后，如果当前 `activeBookId` 不在列表中，则自动选中第一本书：

```tsx
setActiveBookId((prev) => {
  if (prev && bs.some((b) => b.id === prev)) return prev
  return bs[0]?.id ?? ""
})
```

## 四、如何手动验证

### 1. 启动开发服务器

```bash
pnpm dev
```

### 2. 验证初始加载

1. 打开 `http://localhost:3000`
2. 左栏应显示 demo 书籍列表
3. 第一本书应自动被选中（高亮状态）

1. 点击左栏"+ 新建书籍"
2. 应弹出浏览器原生 prompt 输入框

4. 确认后：

   - 中栏应切换到对话模式

## 五、哪些地方仍然使用 mock

| 功能 | 状态 | 说明 |
|------|------|------|
| 书籍列表 | API 优先，mock 回退 | `listBooks()` |
| 创建书籍 | API 优先，mock 回退 | `createBook()` |
| 工作台文件树 | API 优先，mock 回退 | `listWorkbenchTree()` |
| 工作台读文件 | API 优先，mock 回退 | `readWorkbenchFile()` |
| 工作台写文件 | API 优先，mock 回退 | `writeWorkbenchFile()` |
| **章节列表** | **纯 mock** | `listChapters()` 无对应 API，新书创建后章节列表为空（mock 数据中无该 bookId 的章节） |
| **章节内容** | **纯 mock** | 无对应 API |
| **对话消息** | **纯 mock** | `listMessages()` 无对应 API，新书创建后消息列表为空 |
| **ActionPlan** | **纯 mock** | 无对应 API |
| **设定卡片** | **纯 mock** | 无对应 API |
| **试写沙盒** | **纯 mock** | 无对应 API |

**注意**：新创建的书的章节列表和对话消息会显示为空，因为 mock 数据中没有该 bookId 的数据。这是符合预期的行为——后续阶段接入真实 API 后会自动填充。
