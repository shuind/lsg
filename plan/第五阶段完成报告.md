## 二、意图解析规则

### 支持的句式

| 句式 | 示例 | 解析结果 |
|------|------|----------|
| 把X改成男/女的 | 把林晓改成女的 | `{ type: "gender", character: "林晓", target: "女" }` |
| 把X和Y关系改成Z | 把林晓和陈磊关系改成敌对 | `{ type: "relationship", charA: "林晓", charB: "陈磊", relationship: "敌对" }` |

### 支持的关系值

`敌对` → `hostile`，`盟友` → `ally`，`同门` → `sibling`，`师徒` → `master`，`恋人` → `lover`，`朋友` → `friend`，`陌生人` → `stranger`

### 正则表达式

```typescript
// 性别修改
/把\s*(.+?)\s*(?:改|换|变成?)\s*(男|女)\s*(?:的|生)?/

// 关系修改
/把\s*(.+?)\s*(?:和|跟|与)\s*(.+?)\s*(?:的)?关系\s*(?:改|换|变成?)\s*(敌对|盟友|同门|师徒|恋人|朋友|陌生人)/
```

一条消息可包含多个意图（全局扫描）。

## 五、确认执行流程

```
用户点击"应用此节点"
  → POST /api/books/{bookId}/action-plan/confirm { nodeId }
  → confirmNode(bookId, nodeId)
    → loadPendingPlan
    → findNode(plan, nodeId)
    → collectLeaves(node)
    → 对每个 leaf:
      - 如果是性别修改: readBookFile → 替换性别行 → writeBookFile
      - 如果是关系修改: readBookFile(关系图谱.json) → 更新 JSON → writeBookFile
      → writeBookFile 自动写 Ledger + markDirty
    → updateNodeStatus(leaf, "done")
    → updateNodeStatus(node, "done")
    → savePendingPlan
  → 返回 { success, plan }
  → 前端用返回的 plan 刷新右栏
```

### 文件修改规则

**性别修改** (`人物设定/{name}.md`):
- 读取文件内容
- 匹配 `**性别**　...` 行并替换
- 如果文件不存在，创建新文件
- 如果没有性别行，在标题后插入

**关系修改** (`关系图谱.json`):
- 读取 JSON 文件
- 查找 `{charA}_{charB}` 或 `{charB}_{charA}` 键
- 更新值（英文映射）
- 如果键不存在，新增

## 六、新增了哪些 API

| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/api/books/{bookId}/messages` | 解析用户消息，生成 ActionPlan，返回 message + plan |
| GET | `/api/books/{bookId}/action-plan` | 返回当前待确认计划 |

| POST | `/api/books/{bookId}/action-plan/abandon` | 放弃节点，从计划中移除，返回更新后的计划 |

### POST messages 请求/响应

```json
// 请求
{ "content": "把林晓改成女的" }

// 响应
{
  "message": {
    "id": "m-xxx",
    "role": "assistant",
    "content": "已为你拟好变更计划,请在右侧确认。",
    "createdAt": "14:32"
  },
  "plan": [{ ... ActionNode 树 ... }]
}
```

### POST confirm 请求/响应

```json
// 请求
{ "nodeId": "root-xxx" }

// 响应
{ "success": true, "plan": [{ ... 更新后的 ActionNode 树 ... }] }
```
