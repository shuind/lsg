## 二、新增了哪些 API

| 方法 | 路径 | 功能 | 请求体 |
|------|------|------|--------|
| GET | `/api/books` | 返回书籍列表 | — |
| POST | `/api/books` | 创建书籍（含默认目录结构） | `{ title: string }` |
| GET | `/api/books/{bookId}/tree` | 返回书籍文件树 | — |
| GET | `/api/books/{bookId}/file?path=...` | 读取单个文件内容 | — |
| PUT | `/api/books/{bookId}/file` | 写入文件内容 | `{ path: string, content: string }` |

### BookStore 能力

- `listBooks()` — 扫描 `data/books/` 目录，读取所有 `book.json`
- `getBook(bookId)` — 获取单本书元数据
- `createBook(title)` — 创建书籍目录（含 人物设定/、世界观/、章节大纲/、skills/、创作指南.md、关系图谱.json、ledger.jsonl）
- `getBookTree(bookId)` — 递归构建文件树
- `readBookFile(bookId, filePath)` — 读取文件（含路径遍历防护）
- `writeBookFile(bookId, filePath, content)` — 写入文件并更新 book.json 的 updatedAt

## 三、如何手动验证

### 1. 启动开发服务器

```bash
pnpm dev
```

### 2. 验证 API

```bash
# 列出书籍（应返回 demo-guixu）
curl http://localhost:3000/api/books

# 获取文件树
curl http://localhost:3000/api/books/demo-guixu/tree

# 读取文件
curl "http://localhost:3000/api/books/demo-guixu/file?path=人物设定/林晓.md"

# 创建新书
curl -X POST http://localhost:3000/api/books -H "Content-Type: application/json" -d '{"title":"测试书籍"}'

# 写入文件
curl -X PUT http://localhost:3000/api/books/demo-guixu/file -H "Content-Type: application/json" -d '{"path":"测试.md","content":"# 测试"}'
```

### 3. 验证前端

1. 打开 `http://localhost:3000`
2. 左栏书籍列表应显示 demo 数据中的书籍（"归墟之外"等）
3. 点击书籍后，工作台文件树应显示真实文件
4. 点击文件应能读取真实内容
5. 编辑后保存应写入本地文件

### 4. 回退验证

- 停止开发服务器后，前端应自动回退到 mock 数据，不会白屏

## 六、下一步建议

按执行指挥文档的路线，第二阶段应做：

**工作台文件编辑闭环**

1. 文件树点击文件后加载内容（已有 API 支持）
2. 中间编辑器显示 Markdown（已有 textarea）
3. 编辑后保存到对应文件（已有 PUT API）
4. 保存后更新 updatedAt
5. 保存后标记 dirty（为后续向量索引做准备）

这一步基本不需要新 API，只需要把工作台的编辑器与已有 API 真正串联起来。当前工作台的 `readWorkbenchFile` 和 `writeWorkbenchFile` 已经走真实 API，但左栏的章节列表和写作台还是纯 mock，可以一并接入。
