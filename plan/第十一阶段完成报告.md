## 二、messages.jsonl 如何存储

```
data/books/{bookId}/messages.jsonl
```

每条消息一行 JSON，追加写入：

```jsonl
{"id":"m-lx1abc","role":"user","content":"你好","createdAt":"14:32"}
{"id":"m-lx1abd","role":"assistant","content":"主人好，我是 LG…","createdAt":"14:32","brief":{"understood":["主人在询问我的能力"]}}
```

- 文件不存在时返回 `[]`
- 读取时跳过坏行
- 按写入顺序返回

## 三、GET/POST messages 如何持久化

### GET /api/books/{bookId}/messages

```
读取 messages.jsonl → 逐行 JSON.parse → 返回 Message[]
```

### POST /api/books/{bookId}/messages

```
收到 content
  ↓
构造 user message → appendMessages([userMsg])
  ↓
classifyConversationIntent → 处理
  ↓
构造 assistant message → appendMessages([assistantMsg])
  ↓
返回 { message, plan }
```

前端 `handleSend` 仍乐观追加 userMsg（无需等待 API），刷新后 GET 能看到完整历史。

## 四、设定卡如何从真实文件生成

### 人物卡

扫描 `人物设定/*.md`：
- `name` = 文件名去掉 `.md`
- `summary` = 正文前 80-120 字（跳过标题和空行）
- `meta` = 提取 `**性别**`、`**身份**`、`**年龄**` 行
- `category` = `"character"`

### 世界观/规则卡

扫描 `世界观/*.md`：
- 文件名或内容包含"规则""体系""法则" → `category = "rule"`
- 否则 → `category = "place"`
- `name` = 文件名去掉 `.md`
- `summary` = 正文前 80-120 字

### 不生成的

- `关系图谱.json` — 不拆成卡
- `章节大纲/` — 不生成设定卡
- `章节正文/` — 不生成设定卡

## 八、如何手动验证

### 对话历史

1. 启动 dev，打开一本真实书
2. 发送"你好"或"林晓是谁"
3. 刷新页面
4. 对话历史应仍然存在
5. 查看文件：`data/books/{bookId}/messages.jsonl`
6. API 验证：`GET /api/books/{bookId}/messages`

### 设定卡

1. 打开一本有 `人物设定/*.md` 的书
2. 右栏切换到"设定卡"tab
3. 应显示从真实文件生成的人物卡
4. 修改 `人物设定/林晓.md` 后刷新页面，卡片内容应变化
5. API 验证：`GET /api/books/{bookId}/setting-cards`
