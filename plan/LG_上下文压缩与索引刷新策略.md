> 创建时间：2026.05.24 | 状态：设计阶段

## 一、上下文压缩策略

### 1.1 核心矛盾

创作指南 + 当前书籍设定 + 召回内容 + 对话历史，全部塞进一次请求很快超限，截断又会丢失关键信息。

解法：**分层携带，不是压缩所有东西。**

### 1.2 三层上下文

```
┌─────────────────────────────────────────────────────┐
│  Layer A  固定层（每次必带，~800 token 上限）          │
│                                                     │
│  · Skill 注入（见第三章）                             │
│    └── 创作指南_摘要（~500 token，缓存，不每次生成）   │
│  · 当前书籍核心快照                                   │
│    └── 主角列表 / 世界观关键规则 / 当前章节位置         │
│  · 结构化存储，直接注入，不走 LLM 压缩                 │
├─────────────────────────────────────────────────────┤
│  Layer B  召回层（按需带，每次不同）                   │
│                                                     │
│  · 多路召回结果，每条限 200 token                     │
│  · 超过 5 条时按相关性评分截断                        │
│  · 天然按需，无需额外压缩                             │
├─────────────────────────────────────────────────────┤
│  Layer C  对话历史层（滑动窗口）                      │
│                                                     │
│  · 保留最近 N 轮完整对话                              │
│  · N 轮之前：只保留 ActionPlan 执行摘要               │
│    示例："已将林晓性别改为女，关系改为敌对"             │
│  · 具体消息内容丢弃                                   │
└─────────────────────────────────────────────────────┘
```

### 1.3 滑动窗口边界

不按轮数定，**按 token 数定**：

```
构建请求前：
  1. 计算 Layer A + Layer B 的 token 用量
  2. 剩余空间 = 模型上限 - Layer A - Layer B - 预留输出空间
  3. Layer C 从最新一轮往前填，填满为止
  4. 被截断的历史只保留 ActionPlan 摘要行
```

## 二、索引刷新策略

### 2.1 三类索引对比

| 索引类型 | 刷新时机 | 延迟 | 原因 |
|---------|---------|------|------|
| 向量索引 | 延迟，下次对话前增量重建 | 可接受 | embed 代价高，打字时不能频繁触发 |
| 图谱索引 | 实时，ActionExecutor 执行后同步 | 毫秒级 | 程序操作，无 LLM，且后续操作依赖图谱正确性 |
| 时间戳 | 实时，文件写入时更新 last_modified | 毫秒级 | 零成本，直接排序查询 |

### 2.2 向量索引：脏标记 + 增量重建

```
写操作发生
  → 给被修改的文件打上 dirty: true
  → 不立即重建

触发重建的时机：
  用户发起新一轮对话
    → IntentEngine 入口扫描 dirty 文件
    → 有 dirty：对这些文件增量重建向量，清除标记
    → 无 dirty：直接进入意图理解

写作台自动保存（2s 后）
  → 只打 dirty 标记
  → 不触发重建（避免打字时频繁重建）
```

增量重建只对 dirty 文件重新 embed，其余不动。向量库使用支持 upsert 的方案（Qdrant / Weaviate），按文件 id 覆盖更新。

### 2.3 完整数据流

```
用户修改设定
      ↓
ActionExecutor 写文件
      ↓
      ├── 图谱索引：同步更新（实时）
      ├── 向量索引：打 dirty 标记（延迟）
      └── 时间戳：写入 last_modified（实时）

用户下一次发对话
      ↓
IntentEngine 入口
  → 扫描 dirty 文件 → 增量重建向量（~几百毫秒）
  → 意图分析条显示 loading 状态遮住这段延迟
      ↓
正常进入意图理解循环（索引已是最新）
```

## 三、Skill 系统

### 3.1 定位

Skill 系统是用户创作习惯和偏好的结构化沉淀。**本期只做第一个落地形态：创作指南摘要缓存。** 其余 skill 类型后续扩展，数据结构现在定好，不用将来推倒重来。

### 3.2 Skill 数据结构

```typescript
interface Skill {
  id: string
  type: 'style_guide'      // 本期只有这一种
        | 'plot_pattern'   // 未来：惯用情节结构
        | 'char_template'  // 未来：人物创建模板
        | 'world_rule'     // 未来：世界观通用规则

  scope: 'global'          // 跨书籍生效
        | 'book'           // 仅当前书籍生效
  bookId?: string          // scope = book 时必填

  sourceFile: string       // 原始文件路径，e.g. "创作指南.md"
  summaryFile: string      // 摘要缓存路径，e.g. "skills/style_guide_summary.md"
  summaryTokenCount: number

  lastSourceModified: string   // 原始文件最后修改时间
  lastSummaryGenerated: string // 摘要最后生成时间
  dirty: boolean               // 原始文件比摘要新 → 需要重新生成
}
```

### 3.3 本期落地：创作指南摘要

**文件位置：**

```
📚 书名/
├── 创作指南.md          ← 用户手动维护的原始文件
└── skills/
    └── style_guide_summary.md  ← LLM 生成的摘要，自动维护
```

**摘要生成时机：**

```
触发条件（满足任一）：
  · 用户首次引入创作指南
  · 创作指南.md 被修改（检测 last_modified 变化）
  · 用户在工作台手动点击「刷新摘要」

生成方式：
  · 异步任务，不阻塞当前对话
  · 生成完成后更新 summaryFile，清除 dirty 标记
  · 摘要目标长度：500 token 以内
  · 生成提示词重点提取：文风偏好 / 禁忌写法 / 人物塑造原则 / 结构偏好
```

**注入方式：**

每次对话请求构建时，从 `style_guide_summary.md` 读取内容注入 Layer A。读原始文件的情况：用户在工作台查看时。对话上下文永远只用摘要。

**dirty 状态处理：**

```
创作指南.md 被修改
  → skill.dirty = true
  → 下次对话时，意图分析条提示「创作指南已更新，摘要重新生成中...」
  → 异步生成新摘要
  → 本次对话仍使用旧摘要（可接受，摘要滞后一次）
  → 新摘要生成完成后，后续对话自动使用新版
```

### 3.4 未来 Skill 扩展方向（本期不做）

| Skill 类型 | 来源 | 作用 |
|-----------|------|------|
| `plot_pattern` | 从已完成章节中归纳 | 召回时提示"你惯用的三幕结构是..." |
| `char_template` | 用户定义或从现有人物归纳 | 新建人物时预填常用字段 |
| `world_rule` | 从世界观文件中提取硬规则 | 一致性检查时作为约束条件 |

所有 skill 共用同一套 `dirty → 异步重生成 → 注入 Layer A` 的流水线，本期把管道建好，后续只需新增 skill 类型。

## 四、各模块关系总览

```
创作指南.md（用户维护）
      ↓ 修改时触发
  Skill 系统
      ↓ 异步生成
  style_guide_summary.md
      ↓ 每次对话注入
  Layer A（固定层）
      ↓
  IntentEngine
      ↑
  Layer B（召回层）← 向量索引 ← dirty 标记 ← ActionExecutor
      ↑
  Layer C（历史层）← ActionPlan 摘要 ← Ledger
```
