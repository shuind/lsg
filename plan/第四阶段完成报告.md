## 二、LedgerEntry 数据结构

```typescript
interface LedgerEntry {
  id: string           // 唯一 ID，格式: l-{timestamp36}-{random4}
  bookId: string       // 所属书籍 ID
  timestamp: string    // ISO 8601 时间戳
  actor: "user" | "agent"  // 操作者
  action: string       // 操作类型，如 "write_file"
  targetPath: string   // 被操作文件路径
  beforeSnapshot?: string  // 写入前内容（可选）
  afterSnapshot?: string   // 写入后内容（可选）
  summary: string      // 人类可读摘要
}
```

## 三、ledger.jsonl 如何写入和读取

### 存储位置

```
data/books/{bookId}/ledger.jsonl
```

### 写入规则

- 每条日志一行 JSON（JSONL 格式）
- 使用 `fs.appendFile` 追加写入，不覆盖已有内容
- 如果文件不存在，`appendFile` 会自动创建

### 写入时机

在 `writeBookFile()` 中，文件写入完成后自动追加 ledger 条目：

```
writeBookFile(bookId, filePath, content)
  → 读取 beforeSnapshot（文件原内容）
  → 写入文件
  → markDirty
  → 更新 book.json updatedAt
  → appendLedgerEntry（记录 beforeSnapshot + afterSnapshot）
```

### 防递归机制

如果被写入的文件是 `ledger.jsonl` 本身，则跳过 ledger 记录：

```typescript
const isLedgerFile = filePath === "ledger.jsonl"
if (!isLedgerFile) {
  // 读 beforeSnapshot
  // appendLedgerEntry
}
```

### 读取规则

- 使用 `fs.readFile` 读取整个文件
- 按行分割，逐行 `JSON.parse`
- 跳过格式错误的行
- 按 `timestamp` 倒序排列（最新在前）

## 五、工作台 Ledger tab 如何验证

### 1. 启动开发服务器

```bash
pnpm dev
```

### 2. 验证空状态

1. 打开工作台（点击书籍的网格图标）
2. 切换到 Ledger tab
3. 如果该书没有保存过文件，应显示"暂无操作记录"的空状态提示

### 3. 验证记录生成

1. 在编辑器 tab 修改任意文件内容
2. 点击"保存"按钮
3. 切换到 Ledger tab
4. 应看到一条新记录，显示：
   - 时间戳
   - action: `write_file`
   - actor: `user`
   - summary: `手动保存 {文件路径}`
   - targetPath: 被保存的文件路径

### 4. 验证保存后自动刷新

1. 在 Ledger tab 查看当前记录数
2. 切换到编辑器 tab，修改并保存文件
3. 切换回 Ledger tab
4. 应看到记录数增加（通过 key 变化触发重新加载）

### 5. 验证 API

```bash
# 获取某书的 ledger
curl http://localhost:3000/api/books/demo-guixu/ledger

# 直接查看 JSONL 文件
cat data/books/demo-guixu/ledger.jsonl
```

### 6. 验证防递归

```bash
# 手动写入 ledger.jsonl（通过 API）
curl -X PUT http://localhost:3000/api/books/demo-guixu/file \
  -H "Content-Type: application/json" \
  -d '{"path":"ledger.jsonl","content":"test"}'

# 再次获取 ledger，不应出现对 ledger.jsonl 自身的写入记录
curl http://localhost:3000/api/books/demo-guixu/ledger
```
