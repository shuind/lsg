## 三、Chapter / ChapterContent 类型

```ts
// lib/types.ts
export interface Chapter {
  id: string          // encodeURIComponent(文件名去掉.md)
  bookId: string
  title: string       // 取 H1 或文件名
  index: number       // 按文件名排序的序号
  wordCount: number   // 去除空白后的字符数
  status: "draft" | "writing" | "done"
  path: string        // 相对路径: 章节正文/第一章.md
  updatedAt: string   // 文件 mtime ISO
}

export interface ChapterContent {
  id: string
  bookId: string
  title: string
  content: string     // Markdown 全文
  path: string
  updatedAt: string
}
```

## 五、list/get/save/create 章节如何接真实文件

### listChapters(bookId)

扫描 `章节正文/*.md`，按文件名排序。每个文件：
- 提取标题：优先取 `# H1`，否则用文件名去 `.md`
- 计算字数：`content.replace(/\s/g, "").length`
- 状态：wordCount === 0 → draft，否则 → writing
- mtime 通过 `getBookFileMtime` 获取

### createChapter(bookId, title?)

- title 为空时生成 `第{n}章 · 未命名`
- 文件名冲突时追加 `-2`、`-3`
- 初始内容：`# {title}\n\n`
- 通过 `writeBookFile` 写入，自动触发 dirty + Ledger

### getChapter(bookId, chapterId)

- chapterId = `encodeURIComponent(文件名去掉.md)`
- 通过 `readBookFile` 读取
- 返回 `ChapterContent`

### saveChapter(bookId, chapterId, content)

- 通过 `writeBookFile` 保存
- 自动触发 dirty 标记 + Ledger 记录
- 返回最新 `ChapterContent`

## 六、写作台如何保存到本地

```
用户编辑 textarea
  ↓
setContent(e.target.value)
  ↓
useEffect 检测 content 变化
  ↓
debounce 2 秒
  ↓
saveChapter(bookId, chapterId, content)
  ↓
PUT /api/books/{bookId}/chapters/{chapterId}
  ↓
writeBookFile(bookId, path, content)
  ↓
fs.writeFile + markDirty + appendLedgerEntry
```

## 七、dirty / Ledger 是否自动生效

**是。** 因为 `saveChapter` 通过 `writeBookFile` 写入，而 `writeBookFile` 自动：
- 调用 `markDirty(bookId, filePath)` 标记脏文件
- 调用 `appendLedgerEntry` 写入 ledger.jsonl
- 更新 `book.json` 的 `updatedAt`

可通过以下方式验证：
```bash
# 查看 dirty files
curl http://localhost:3000/api/index/dirty-files

# 查看 ledger
curl http://localhost:3000/api/books/{bookId}/ledger
```

## 十、如何手动验证

1. 打开页面，选择一本真实书（如 demo-guixu）
2. 如果该书有 `章节正文/*.md`，左栏章节列表应显示真实章节
3. 点击章节，进入写作台，应显示真实正文
4. 修改正文，等待 2 秒自动保存
5. 打开工作台 Ledger tab，应看到章节保存记录
6. 查询 dirty files（GET /api/index/dirty-files），应看到章节正文文件
7. 点击新建章节，应在 `data/books/{bookId}/章节正文/` 下创建新的 `.md` 文件
8. 新章节应出现在左栏，点击后可编辑保存
9. 章节不存在时，写作台应显示"章节不存在"提示，不白屏
