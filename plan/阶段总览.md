# LG 项目阶段总览

## 第一阶段 — 项目骨架

搭建 Next.js 项目,配置 Tailwind CSS、shadcn/ui 组件库。建立基础目录结构和主题系统。

## 第二阶段 — UI 框架

实现三栏布局:左侧边栏(书籍/章节列表)、中间主区、右侧边栏(设定卡片)。完成基础导航和状态切换。

## 第三阶段 — 对话面板

实现聊天界面,支持用户发送消息和 AI 回复。消息列表、输入框、意图理解卡片的基础 UI。

## 第四阶段 — Action Plan

实现操作计划的树状展示。Agent 返回的操作步骤以卡片形式呈现,支持确认和放弃。

## 第五阶段 — 写作台

独立写作界面,带富文本工具栏、正文编辑区、字数统计。章节从列表点击进入写作台。

## 第六阶段 — 工作台

文件树浏览功能。以树状结构展示书籍目录下所有文件,支持点击查看和编辑。

## 第七阶段 — 设定卡片与检索

右侧边栏展示设定卡片(人物、世界观)。实现基于关键词的上下文检索。

## 第八阶段 — LLM 接入

接入 OpenAI 兼容 API,支持 MiMo 和 DeepSeek 两个 Provider。实现 `callChatCompletion` 统一调用层。

## 第九阶段 — 对话 Agent 重构

重构对话流程:意图分类 → 路由 → 响应。Agent 自称"LG",称呼用户为"主人"。意图理解卡片持久显示,不消失。

四种意图类型:
- `meta` — 元问题(你是谁、你能做什么)
- `book_query` — 查询类(角色关系、剧情问题)
- `book_mutation` — 修改类(改设定、写章节)
- `clarification_needed` — 需要澄清

## 第十阶段 — 真实章节系统

章节从 mock 数据改为真实文件存储。每个章节对应 `章节正文/*.md` 文件。实现完整 CRUD API:
- `listChapters` — 扫描目录
- `createChapter` — 创建文件
- `getChapter` — 读取内容
- `saveChapter` — 保存内容(触发 dirty + ledger)

## 第十一阶段 — 对话持久化与设定卡片真实化

对话历史存储到 `messages.jsonl`,每次启动恢复历史。设定卡片从真实文件扫描生成:
- `人物设定/*.md` → 人物卡片(提取性别、身份、年龄)
- `世界观/*.md` → 世界观卡片

## 第十二阶段 — 试写沙盒 AI 续写

写作台底部增加试写沙盒。调用 LLM 基于当前章节上下文和创作指南生成 300-600 字续写。
- 试写不写入正文,不触发 dirty/ledger
- 用户点击"保留到草稿"后才追加到正文
- "继续生成"将已有 draft 作为上下文继续续写
- 无 API key 时返回 fallback 文本
