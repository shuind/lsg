## 二、新增文件详解：conversation-agent.ts

### 导出项

| 导出 | 类型 | 用途 |
|------|------|------|
| `LG_AGENT_SYSTEM_PROMPT` | `string` | LG Agent 系统提示词，定义角色和能力 |
| `ConversationIntent` | `"meta" \| "book_query" \| "book_mutation" \| "clarification_needed"` | 意图类型 |
| `classifyConversationIntent(text)` | `(string) → ConversationIntent` | 意图分类（纯规则，不调 LLM） |
| `replyToMetaQuestion(text)` | `(string) → string` | meta 消息的固定回复 |
| `answerBookQueryWithLlm(bookId, query)` | `(string, string) → Promise<string \| null>` | 用 LLM 回答书籍查询 |
| `fallbackAnswerFromContexts(query, contexts, summary)` | `(string, contexts, string) → string` | 无 LLM 时的上下文兜底回答 |

### classifyConversationIntent 分类逻辑

```
输入文本
  ↓
正则匹配 meta？ → "meta"（问候、能力询问、确认词）
  ↓
正则匹配修改指令？ → "book_mutation"（把X改成Y、改一下设定）
  ↓
正则匹配模糊指令？ → "clarification_needed"（"改一下"无对象）
  ↓
正则匹配查询词？ → "book_query"（谁、什么、哪里、人物、关系…）
  ↓
默认 → "book_query"
```

### answerBookQueryWithLlm 流程

```
query
  ↓
并行: retrieveContext(bookId, query) + readStyleGuideSummary(bookId)
  ↓
组装上下文块（文件名 + 摘要，不暴露路径）
  ↓
callChatCompletion(config, [system: LG_AGENT_PROMPT, user: 上下文 + 问题])
  ↓
返回自然语言回答 或 null（失败时）
```

## 三、messages route 重构后的流程

```
POST /api/books/[bookId]/messages
  ↓
classifyConversationIntent(content)
  ↓
  ├─ "meta"
  │   → replyToMetaQuestion(content)
  │   → 返回 message（无 retrieval，无 LLM，无 plan）
  │
  ├─ "clarification_needed"
  │   → 返回固定提示："你的指令不够明确…"
  │   → 返回 message（无 retrieval，无 LLM，无 plan）
  │
  ├─ "book_query"
  │   → answerBookQueryWithLlm(bookId, content)
  │     ├─ 成功 → 返回 LLM 自然语言回答（无 contextPaths 暴露）
  │     └─ 失败 → retrieveContext + readStyleGuideSummary
  │              → fallbackAnswerFromContexts（上下文兜底）
  │   → 返回 message（无 plan，无 contextPaths）
  │
  └─ "book_mutation"
      → 原有 IntentEngine 流程（不变）
      → retrieveContext → readStyleGuideSummary → LLM/规则解析
      → 返回 message（含 brief: understood/contextPaths/missing）+ plan
```

## 四、与第八阶段的关键差异

| 对比项 | 第八阶段 | 第九阶段 |
|--------|----------|----------|
| meta 检测 | 本地 `isMetaOrGreetingMessage` | `classifyConversationIntent` 统一分类 |
| 书籍查询 | 不支持（只能改，不能问） | `book_query` 分支：LLM 自然语言回答 |
| contextPaths | 所有回复都可能暴露 | `book_query` 和 `meta` 不暴露路径 |
| 模糊指令 | 走规则解析（无 plan） | `clarification_needed` 明确提示补充信息 |
| LLM 复用 | `callLlm` 内部函数 | `callChatCompletion` 导出，供两个模块共用 |
| 系统提示 | 固定在 `callLlm` 内 | `LG_AGENT_SYSTEM_PROMPT` 独立导出 |

## 六、验证场景

### 1. meta 消息（不触发 retrieval）

```bash
curl -X POST http://localhost:3000/api/books/demo-guixu/messages \
  -H "Content-Type: application/json" \
  -d '{"content":"你好"}'
# → 固定问候回复，无 plan

curl -X POST http://localhost:3000/api/books/demo-guixu/messages \
  -H "Content-Type: application/json" \
  -d '{"content":"你是谁"}'
# → 自我介绍回复，无 plan
```

### 2. book_query（LLM 回答，无 contextPaths 暴露）

```bash
curl -X POST http://localhost:3000/api/books/demo-guixu/messages \
  -H "Content-Type: application/json" \
  -d '{"content":"林晓是谁"}'
# → LLM 根据上下文回答，message.brief 无 contextPaths

curl -X POST http://localhost:3000/api/books/demo-guixu/messages \
  -H "Content-Type: application/json" \
  -d '{"content":"谁跟谁是敌对关系"}'
# → LLM 回答关系信息
```

### 3. book_mutation（IntentEngine，不变）

```bash
curl -X POST http://localhost:3000/api/books/demo-guixu/messages \
  -H "Content-Type: application/json" \
  -d '{"content":"把林晓改成女的"}'
# → 生成 plan，返回 brief（含 understood/contextPaths）
```

### 4. clarification_needed

```bash
curl -X POST http://localhost:3000/api/books/demo-guixu/messages \
  -H "Content-Type: application/json" \
  -d '{"content":"改一下"}'
# → 提示补充信息，无 plan
```

### 5. 无 API key 时 book_query 回退

```bash
# 不设置 API key
curl -X POST http://localhost:3000/api/books/demo-guixu/messages \
  -H "Content-Type: application/json" \
  -d '{"content":"林晓是谁"}'
# → fallbackAnswerFromContexts 返回上下文摘要
```
